API Reference

Pipeline

`minillm.pipeline.main`

: Entry point that wires together retrieval, memory and LLM calls.

It constructs a `SimpleOrchestrator` with nodes for document retrieval, prompt rendering, the LLM call and post-processing steps such as memory updates.

: Example entry point that wires together retrieval, memory and LLM calls.

LLMClient

`minillm.llm_client.LLMClient`

: Abstraction for calling different language model providers. The default provider is Gemini. The `generate` method sends prompts to Gemini's API and returns the resulting text.

Embeddings

`minillm.embeddings.gemini_embed`

: Helper that invokes Gemini's embedding endpoint. Both vector and graph retrievers use this function to compute text embeddings.

: Abstraction for calling different language model providers. Defaults to Gemini.

Retrievers

`minillm.retriever.VectorRetriever`

: FAISS-based vector search that stores embeddings on disk.

`minillm.retriever.GraphRetriever`

: Neo4j graph retrieval that performs similarity search over connected entities.

Memory Manager

`minillm.memory_manager.MemoryManager`

: Persists message history and generates conversation summaries for long-term context.

: FAISS-based vector search using Gemini embeddings.

`minillm.retriever.GraphRetriever`

: Neo4j graph retrieval using embeddings for similarity.
